# Document Management System - Login Testing Project
QA manual testing project for Login functionality with test cases, bug reports, and test plan

## 📌 Project Overview
This project is a simple Document Management System (DMS) focusing on the **Login module**.

The goal of this project is to demonstrate **manual testing skills**, including:
- Analyzing requirements
- Designing test cases
- Executing test cases
- Reporting bugs

---

## 🎯 Objectives
- Verify login functionality using email and password
- Validate user input (email format, password rules)
- Ensure system behavior is correct under different scenarios
- Identify and report defects

---

## 🧪 Testing Scope
The following features are tested:

- Login with valid credentials
- Login with invalid credentials
- Email validation
- Password validation:
  - Minimum length (8 characters)
  - Maximum length (255 characters)
  - Uppercase & lowercase requirement
- Account lock after 5 failed attempts
- Remember Me functionality
- Forgot Password navigation

---

## 🧩 Test Types
- Positive Testing
- Negative Testing
- Boundary Testing
- Validation Testing

## 📊 Test Artifacts

### ✔ Test Cases
- Stored in: `testing/testcase.xlsx`
- Include:
  - Test steps
  - Expected vs Actual results
  - Status (Pass/Fail)
  - Bug ID (if failed)

---

### 🐞 Bug Report
- Stored in: `testing/bug_report.md`
- Includes:
  - Bug ID
  - Steps to reproduce
  - Expected vs Actual result
  - Severity & Priority

---

### 📄 Test Plan
- Stored in: `testing/test_plan.md`
- Describes:
  - Scope
  - Test types
  - Environment
